基于Redis-shake的redis集群数据迁移

Created2024-09-08|Updated2024-09-08|redis
|Post Views:

1 实验环境

Redis Version:6.2.14

Redis Cluster Installer: redis-console-manager-0.2.3-20240905

Source os: Red Hat Enterprise Linux Server release 7.9 (Maipo)

Target os:Red Hat Enterprise Linux release 8.5 (Ootpa)

Source Redis Cluster：192.168.101.15/17/19：16380/17380(ip:port)

Target Redis Cluster：192.168.101.20/205/206：16380/17380(ip:port)

2 环境搭建

2.1 Redis-cluster

在源服务器上192.168.101.15/17/19以及目标服务器192.168.101.20/205/206上搭建redis-cluster集群，使用redis-console-manager-0.2.3-20240905版本安装器，具体操作见《Redis内存数据库通⽤安装器》。使用./redis-cli -h <ip> -p <port> -a <auth>连接集群节点，并使用cluster nodes命令，若见下图结果，即redis cluster搭建完成。

2.2 Redis-shake

在源服务器上任选一台，选择任一目录(以/root/目录为例)，使用wget ​https://github.com/tair-opensource/RedisShake/releases/download/v4.1.1/redis-shake-linux-amd64.tar.gz​获取安装包，若出现Unable to establish SSL connection.错误，可以跳过ssl验证获取安装包，即执行wget --no-check-certificate ​https://github.com/tair-opensource/RedisShake/releases/download/v4.1.1/redis-shake-linux-amd64.tar.gz​。获取成功过后，使用tar -xzvf命令解压，在该目录下存在redis-shake执行程序和shake.toml配置文件样例。

3 模拟测试

3.1 Redis-shake sync模式

3.1.1 Sync模式介绍

当源数据库兼容 PSync 协议时，推荐使用sync_reader。兼容 PSync 协议的数据库有Redis,Tair, ElastiCache 部分兼容, MemoryDB 部分兼容(以Sync Reader | RedisShake (tair-opensource.github.io)为准)。

Sync模式支持全量同步和增量同步，增量同步在全量同步完成后自动开始。如果源redis是集群，要启动多个Redis-Shake，从不同的redis实例中拉取数据，同时源端不能开启move slot功能。

3.1.2 Sync配置文件参数介绍

配置redis-shake sync.toml文件时，其主体由两部分组成：reader/writer，且两部本内容几乎相同。下面以网站Sync Reader | RedisShake (tair-opensource.github.io)给出reader部分写法为实例，介绍各参数作用以及参数值。

参数名 参数值
cluster True，源端为redis集群，false反之
address 源端地址,如为redis集群，address选取一个节点即可，例如address = “192.168.101.15:16380”
Username/password Redis登录用户及密码
tls 源端是否开启TLS/SSL，不需要配置证书因为RedisShake没有校验服务器证书
sync_rdb 是否同步RDB，设置为false时，RedisShake会跳过全量同步阶段
sync_aof 是否同步AOF，设置为false时，RedisShake会跳过增量同步阶段，此时RedisShake会在全量同步阶段结束后退出

3.2 Redis-shake scan模式

3.2.1 Scan模式介绍

当云厂商出于种种考虑禁用了 Redis 的 PSync 命令时，使用sync进行数据迁移无法使用，此时可使用scan模式实现数据迁移。原理是调用 scan 命令来获取 Redis 中的 key，然后使用 dump 命令获取 key 的内容，最终使用 restore 命令恢复 key 至目的端。scan_reader 的性能与数据一致性均不如 sync_reader，应尽可能选择 sync_reader。

3.2.2 Scan配置文件参数介绍

同上述内容，下面以网站Sync Reader | RedisShake (tair-opensource.github.io)给出reader部分写法为实例，介绍各参数作用以及参数值。

参数名 参数值
cluster True，源端为redis集群，false反之
address 源端地址,如为redis集群，address选取一个节点即可，例如address = “192.168.101.15:16380”
tls 源端是否开启TLS/SSL，不需要配置证书因为RedisShake没有校验服务器证书
dbs 源端为非集群模式时，支持仅同步指定DB库
scan 是否开启SCAN阶段，设置为false时，RedisShake会跳过全量同步阶段
ksn 开启ksn参数后，RedisShake会订阅源端的Key变化，实现增量同步
count 全量同步时每次从源端拉取的key的个数，默认为1，改为较大值可以显著提升同步效率，同时也会提升源端压力
