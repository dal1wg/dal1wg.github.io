Redis4.0.11 Cluster搭建及出现问题

Created2024-10-21|Updated2025-02-13|redis
|Post Views:

一.系统环境

os：Red Hat Enterprise Linux Server release 7.9 (Maipo)

Redis_version: 4.0.11

OpenSSL_version:1.1.1t 7 Feb 2023

server:192.168.101.15/17/19

二.集群规划

将集群部署为三主三从的cluster集群

server master slave
192.168.101.15 16380 17380
192.168.101.17 16380 17380
192.168.101.19 16380 17380

三.部署过程

（1）各节点运行实例

本次集群部署过程中，在每个服务器创建了测试用户test2，避免在root目录下与其他的文件混淆

首先获取redis4.0.11版本源码wget https://download.redis.io/releases/redis-4.0.11.tar.gz​，然后make​编译一下

将编译好的文件移至公司redis_all/bin目录下，然后tar zcvf filename file​压缩一下

使用redis-console-manager-x86_64集群部署工具将其部署到各服务器上，可以发现的是公司集群搭建工具在执行redis-cli create –cluster时出现错误，这是由于redis-cli4.0.11不支持 --cluster​字段，但此时各节点实例正在运行

（2）配置ruby环境

在此引用centos7/rhel7下安装redis4.0集群 - Adrian·Ding - 博客园文章，进行ruby环境的部署，并配置好ruby redis依赖，即可正常运行redis-trib.rb文件

（3）配置redis cluster集群

执行redis-trib.rb create --replicas 1 192.168.101.15:16380 192.168.101.17:16380 192.168.101.19:16380 192.168.101.15:17380 192.168.101.17:17380 192.168.101.19:17380​即可创建redis 集群

​​

三.相关问题

1.OpenSSL

为什么我会在系统环境中添加openssl版本，因为在ruby环境的搭建过程中由于openssl与ruby不兼容，导致ruby的编译一直报错！！

由于在该服务器上之前编译redistimeseries，需要的openssl版本要达到3.X以上，且在redistimeseries模块运行时也会调用libssl.so.3文件，于是在该服务器上的Openssl版本一直为3.X，但、在ruby环境部署的过程中，进行编译，openssl3.x对于部署的ruby2.5.5版本过高，有需要重新降低openssl版本，对此人麻了。。。。

2.[ERR] Sorry, can’t connect to node 192.168.101.15:16380

这个报错忘记由于什么事件导致的，但重新每个节点就好了
